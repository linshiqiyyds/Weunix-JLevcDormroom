WeUnix Kismet7 使用说明

一、启动方式

1. 双击运行：
   E:\Weunix\desktop\gui\src-tauri\target\release\weunix-desktop.exe

2. 首次启动后，等待顶部状态显示 Live。
   如果提示后端离线，先关闭程序再重新打开。

二、推荐使用流程

1. 进入“导入”页面。
2. 点击“一键自动抓取”。
3. 在 PC 客户端进入目标功能页面，等待 WeUnix 捕获 wxopid。
4. 捕获成功后，账号会自动加入本地配置。
5. 回到“账号”或“总览”页面，点击“同步”。
6. 同步完成后，“个人信息”区域会显示姓名、组织、分组、编号、角色和批次时间。
7. 点击“体检”确认自动登录、Token、身份信息、批次和资源接口是否正常。
8. 点击“演练”做安全预检查，不会提交确认单。
9. 确认无误后，再按需要点击“启动”或“启动全部”。

三、页面说明

总览：
- 查看账号数量、运行任务、资源快照、偏好和事件流。
- 顶部短句来自在线文案接口，失败时会自动使用本地兜底文案。

账号：
- 每个账号都有独立卡片。
- “个人信息”会在同步后刷新。
- “资源快照”展示最近一次同步或体检拿到的资源数量。
- “复制诊断”会复制一段适合发给维护者的排查文本。

导入：
- 小白用户优先使用“一键自动抓取”。
- 高级用户可以粘贴包含 wxopid 的请求、链接或 cURL。
- 授权 code 通常是一次性的，不建议长期依赖。

日志：
- 查看同步、体检、演练、启动、停止、错误等运行记录。

设置：
- 设置开放时间、偏好、敏感信息显示方式。
- 开放时间不用手写，支持“立即 / 今天 / 明天 / 自定义”。
- 可以备份配置，也可以恢复上一次配置。

文档：
- 程序内置“文档”页，适合第一次使用的人按步骤查看。

四、敏感信息显示

默认隐藏敏感信息：
- 姓名显示为脱敏形式。
- 组织、分组、角色、批次显示“已同步”。
- 编号、UserId、openid 只显示前后片段。

需要核对完整信息时：
- 在账号卡片点击“临时核对”，或进入“设置 / 执行策略”区域切换显示方式。
- 查看完建议再切回隐藏。

五、账号管理

每个账号卡片都支持：
- 编辑备注名，例如“本人 / 张三 / 备用”。
- 编辑标签，例如“主号 / 备用”。
- 复制摘要，按照当前隐私模式生成干净文本。
- 复制诊断，适合发给维护者排查。
- 删除账号。删除前会弹出二次确认；运行中账号会先停止再移除。

六、常见问题

1. 点击“一键自动抓取”后没有捕获
   请确认 PC 客户端确实进入了目标功能页面。只打开二维码或授权入口通常拿不到永久 opid。

2. 捕获到了但添加失败
   可能是 opid 不可用、已过期、账号未绑定或接口拒绝。可以复制诊断发给维护者。

3. 同步后资源数量是 0
   这不一定是错误，可能是接口正常但当前无可用资源。可以等待开放时间或调整偏好。

4. 页面文字模糊
   源码仓库不内置商业/系统字体文件。程序会优先使用系统可用字体；若仍模糊，可检查 Windows 缩放和显示器缩放设置。

5. 网络突然异常
   点击“停止并恢复”，WeUnix 会恢复启动前的系统代理设置。

七、注意事项

- 不要把完整 openid、UserId、诊断日志随意公开。
- 删除账号前会有二次确认，但仍建议定期备份配置。
- 长时间运行前，请先执行一次同步和体检。
- 程序是否成功取决于账号有效性、服务状态、开放时间、资源余量和接口返回。
