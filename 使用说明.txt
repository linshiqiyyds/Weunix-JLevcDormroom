WeUnix 小白使用说明

重要提醒：
本项目仅供学习交流使用，不提供任何技术支持或可用性承诺。
不要公开登录标识、UserId、支付链接、二维码截图或完整诊断日志。

一、第一次打开

1. 双击 WeUnix 安装包完成安装。
2. 从桌面或开始菜单打开 WeUnix。
3. 等待顶部状态显示 Live。
4. 如果显示后端离线，先关闭软件，再重新打开。
5. 仍然失败时，进入“日志”页，把错误截图或诊断发给维护者。

二、最推荐的使用顺序

请按这个顺序操作，不要跳步骤：

导入账号 -> 同步个人信息 -> 体检 -> 演练 -> 设置时间和偏好 -> 启动 -> 扫码支付

三、导入账号

小白推荐：

1. 点击左侧“导入”。
2. 点击“一键自动抓取”。
3. 打开 PC 客户端。
4. 在 PC 客户端里进入对应业务页面。
5. 不要只停在二维码页或授权页，要真正进入功能页面。
6. 等待 WeUnix 提示捕获成功。
7. 捕获成功后，账号会自动加入账号列表。

如果抓取后网络异常：

1. 回到“导入”页。
2. 点击“停止并恢复”。
3. 关闭 WeUnix 后重新打开。

高级手动方式：

把能识别出登录标识的链接、请求文本或 cURL 粘贴到导入框。
注意：扫码 code 通常是一次性的，不适合作为长期账号使用。

四、同步个人信息

1. 点击左侧“账号”。
2. 找到刚添加的账号卡片。
3. 点击“同步”。
4. 等按钮 loading 结束。
5. 查看“身份资料”区域。

同步后应该能看到姓名、组织、分组、编号、角色和批次时间。
如果开启隐私模式，部分内容会显示为“已同步”或打码，这是正常的。

五、设置偏好和时间

房型偏好：

1. 进入“总览”或“设置”。
2. 找到执行策略区域。
3. 选择 4人间、8人间、都可以或不限。
4. 默认是 4人间。如果不确定，也可以改成“4人间或8人间”。

执行时间：

1. 选择“立即 / 今天 / 明天 / 自定义”。
2. 用下拉框选择小时、分钟、秒。
3. 不要手写时间格式。
4. 保存策略。

六、体检

体检不会提交订单，可以放心点击。

1. 在账号卡片点击“体检”。
2. 等待结果。
3. 查看自动登录、Token、身份信息、批次、资源和偏好匹配。

如果资源数量是 0，不一定是错误。可能只是还没开放。

七、演练

演练不会提交订单。

1. 同步成功后点击“演练”。
2. 看资源数量和匹配结果。
3. 如果演练失败，先不要启动。
4. 点击“复制诊断”发给维护者。

八、正式启动

启动前确认：

1. 账号已经导入。
2. 个人信息已经同步。
3. 房型偏好已经保存。
4. 时间策略已经保存。
5. 体检没有明显错误。
6. 演练结果符合预期。

启动方式：

单个账号：在账号卡片点“启动”。
多个账号：在总览页点“启动全部”。

启动后不要反复点按钮。

九、抢到订单后

程序逻辑是：

请求到一次真实订单 -> 停止继续抢单请求 -> 显示支付二维码 -> 进入支付状态追踪

这样可以避免重复提交和重复锁单。

如果订单成功，账号卡片会显示：

1. 订单号。
2. 金额。
3. 支付倒计时。
4. 微信扫码二维码。
5. 打开支付链接。
6. 复制支付链接。

价格不是写死的，程序会优先读取接口返回金额，也能识别标题里的金额。

十、支付

看到二维码后：

1. 打开微信。
2. 扫描 WeUnix 里的二维码。
3. 在微信里确认金额。
4. 金额无误后支付。
5. 支付后不要马上关闭 WeUnix。
6. 等待订单状态自动刷新。

程序不能替你绕过微信支付确认，微信里的付款必须由用户本人完成。

十一、支付后

如果支付成功，订单区会更新支付状态、房间号、床位号和支付时间。

如果已经付款但 WeUnix 还显示待支付：

1. 等 10 到 30 秒。
2. 点击账号卡片的“同步”。
3. 点击“复制订单”。
4. 把订单摘要发给维护者。

十二、隐私模式

默认隐藏敏感信息。

需要核对账号时，可以点“临时核对”。
看完建议切回隐藏。

截图、发朋友圈、发群、发 GitHub 前，一定确认敏感信息已经打码。

十三、账号管理

每个账号都可以：

1. 设置备注，例如本人、张三、备用。
2. 设置标签，例如主号、备用。
3. 复制摘要。
4. 复制诊断。
5. 删除账号。

删除账号会二次确认。如果账号正在运行，会先停止再删除。

十四、配置备份

建议在这些时候备份：

1. 第一次成功添加账号后。
2. 批量添加账号后。
3. 正式使用前。
4. 修改偏好或时间前。

误删账号时，可以尝试恢复上一次配置。

十五、软件更新

1. 进入“设置”。
2. 点击“检查更新”。
3. 如果有新版本，点击“下载并安装”。
4. 安装完成后点击“重启生效”。

如果自动更新失败，可以打开 GitHub 或 Gitee 页面手动下载最新版安装包。

十六、出问题时怎么反馈

请说明：

1. 你点到了哪一步。
2. 页面显示什么状态。
3. 是否同步成功。
4. 是否体检。
5. 是否演练。
6. 复制诊断文本。
7. 如果是支付问题，再复制订单摘要。

不要发送：

1. 完整登录标识。
2. 完整 UserId。
3. 支付二维码截图。
4. 支付链接。
5. 身份证、手机号等个人敏感信息。

最短口诀：

先导入，再同步，再体检，再演练。
确认时间和偏好，最后启动。
出二维码就微信扫码。
出问题就复制诊断。
