# WeUnix 小白使用说明书

这份说明写给第一次使用 WeUnix 的人。你不需要会编程，也不需要看懂接口、Token、日志这些词。请按顺序操作，不要跳步骤。

> 重要提醒：本项目仅供学习交流使用，不提供任何技术支持或可用性承诺。请不要公开自己的登录标识、UserId、完整诊断日志、付款链接或二维码截图。

---

## 0. 你先知道它是干什么的

WeUnix 是一个本地桌面控制台，用来把寝室选择流程整理成几个按钮：

1. 添加账号。
2. 同步个人信息，确认这个账号是谁。
3. 体检，确认登录和接口是否正常。
4. 演练，确认当前偏好和资源能否匹配，但不会提交。
5. 正式启动，等待开放并尝试创建订单。
6. 如果创建订单成功，软件会显示支付二维码。
7. 用户用微信扫码支付，软件继续追踪订单状态。

程序会尽量减少小程序页面白屏和反复加载带来的等待。根据实测，关键流程体感卡顿可降低约 82%，但最终效果仍取决于网络、账号状态、开放时间、资源余量和服务端返回。

## 1. 安装和启动

### 1.1 下载哪个文件

普通用户只需要下载安装包：

```text
WeUnix_版本号_x64-setup.exe
```

或者：

```text
WeUnix_版本号_x64_zh-CN.msi
```

如果你不知道选哪个，优先选 `setup.exe`。

### 1.2 安装

1. 双击安装包。
2. 如果 Windows 提示“是否允许此应用更改设备”，点“是”。
3. 安装完成后，从桌面或开始菜单打开 `WeUnix`。

### 1.3 第一次打开要看哪里

打开后，先看右上角或顶部状态：

- 显示 `Live`：正常。
- 显示后端离线、Backend offline、红色错误：关闭软件，再重新打开一次。
- 如果仍然失败：不要乱点，进入“日志”页，把错误截图或复制诊断给维护者。

---

## 2. 第一次使用的正确顺序

请按这个顺序来：

```text
导入账号 -> 同步 -> 体检 -> 演练 -> 设置时间和偏好 -> 启动 -> 扫码支付
```

不要一上来就点“启动”。如果没有同步、没有体检，出错时你不知道是账号问题、网络问题、时间问题，还是资源没开放。

---

## 3. 导入账号

### 3.1 小白推荐：一键自动抓取

1. 打开 WeUnix。
2. 点击左侧 `导入`。
3. 点击 `一键自动抓取`。
4. 软件会提示正在启动抓取。
5. 打开 PC 客户端。
6. 在 PC 客户端里进入对应业务页面。
7. 不要只停留在二维码页或授权页，要真正进入功能页面。
8. 等 WeUnix 显示捕获成功。
9. 成功后账号会自动添加到账号列表。
10. 如果提示已经添加过，说明这个账号本地已经存在。

注意：

- 一键自动抓取会临时接管系统代理。
- 捕获完成或点击停止后，会恢复代理。
- 如果你发现浏览器打不开网页，回到导入页点击 `停止并恢复`。

### 3.2 高级用户：手动粘贴

如果你懂抓包，可以把能识别出登录标识的链接、请求文本或 cURL 粘贴到导入框。

你需要的是长期可用的账号登录标识，不是一次性扫码 code。

常见错误：

- 扫码 code：通常只能用一次，不适合作为账号长期使用。
- `appid`：不是账号。
- 复制授权页面链接：通常拿不到长期账号标识。

---

## 4. 同步个人信息

添加账号后，请马上同步。

操作步骤：

1. 点击左侧 `账号`。
2. 找到刚添加的账号卡片。
3. 点击 `同步`。
4. 等按钮 loading 结束。
5. 查看“身份资料”区域。

同步后你应该看到：

- 姓名
- 组织
- 分组
- 编号
- 角色
- 批次时间

如果你看不到完整信息，可能是隐私模式开启。隐私模式下会打码，这是正常的。需要核对时点 `临时核对`，看完再切回隐藏。

如果一直是“待同步”：

1. 点账号卡片里的 `复制诊断`。
2. 把诊断发给维护者。
3. 不要公开完整登录标识或 UserId。

---

## 5. 设置偏好和时间

### 5.1 设置房型偏好

进入 `总览` 或 `设置`，找到执行策略区域。

常见选项：

- 4人间、8人间：两种都可以。
- 4人间：优先匹配人数更少、价格相对更高的房型。
- 8人间：优先匹配人数更多、价格相对更低的房型。
- 不限：服务端返回什么可用资源就按程序策略选择。

默认偏好是 `4人间`。如果你不确定，也可以改成 `4人间、8人间`。

### 5.2 设置执行时间

不要手写时间，使用下拉选择：

- 立即：点启动后马上执行。
- 今天：选择今天的小时、分钟、秒。
- 明天：选择明天的小时、分钟、秒。
- 自定义：先选日期，再选时间。

设置完后点保存。保存成功后页面会提示当前生效时间。

如果你不知道具体开放时间，可以先选“立即”，在接近开放时手动启动。

---

## 6. 体检

体检不会提交订单，放心点。

操作步骤：

1. 进入 `账号`。
2. 找到账号卡片。
3. 点击 `体检`。
4. 等待结果。

体检会检查：

- 自动登录是否通过。
- Token 是否拿到。
- 学生信息是否同步。
- 批次是否开放。
- 房源接口是否正常。
- 当前偏好是否能匹配资源。

如果房源数量是 0，不一定是错误。可能是当前还没开放，或者接口正常但暂时无可选资源。

---

## 7. 演练

演练也不会提交订单。

它的作用是：正式启动前再走一遍关键检查，看看账号、偏好和接口状态是否合理。

操作步骤：

1. 同步成功后，点击 `演练`。
2. 看资源数量和匹配状态。
3. 如果演练失败，先不要启动。
4. 复制诊断给维护者。

---

## 8. 正式启动

确认下面几项都没问题后再启动：

- 账号已经导入。
- 个人信息已经同步。
- 房型偏好已经保存。
- 时间策略已经保存。
- 体检没有明显错误。
- 演练结果符合预期。

启动方式：

- 单个账号：在账号卡片点 `启动`。
- 多个账号：在总览页点 `启动全部`。

启动后不要反复点按钮。按钮会显示运行状态。程序会根据你的时间策略等待或立即执行。

---

## 9. 如果抢到订单会发生什么

当程序发现可用资源并成功创建订单后，会立刻停止继续抢单请求。

也就是说：

```text
请求到一次真实订单 -> 停止继续请求房源和下单 -> 显示支付二维码 -> 进入支付状态追踪
```

这样做是为了避免重复提交、重复锁单或造成混乱。

如果订单成功，账号卡片会出现：

- 订单状态
- 订单号
- 金额
- 支付倒计时
- 微信扫码二维码
- 打开支付链接
- 复制支付链接

价格不是写死的。程序会优先读取接口返回的金额；如果接口把金额写在标题里，例如 `4人间(剩余12间) ¥价格`，程序也会自动解析。只有在接口没有明确返回房型文字时，程序才会把同一批房源里价格较高的房型作为 4人间兜底，把价格较低的房型作为 8人间兜底。

---

## 10. 支付

看到二维码后：

1. 打开微信。
2. 扫描 WeUnix 里显示的二维码。
3. 在微信里确认金额。
4. 确认无误后支付。
5. 支付后不要马上关闭 WeUnix。
6. 等 WeUnix 自动刷新订单状态。

程序会在订单创建后立刻查一次支付状态，前 30 秒会更快地轮询，之后降低频率继续确认。

如果二维码扫不了：

1. 点击 `打开链接`。
2. 如果仍然不行，点击 `复制链接`。
3. 把链接粘贴到浏览器或微信能打开的位置。

注意：程序不能替你绕过微信支付确认。微信里的付款确认必须由用户本人完成。

---

## 11. 支付成功后看哪里

支付成功后，账号卡片里的订单区域会更新。

你可能看到：

- 支付成功
- 房间号
- 床位号
- 支付时间
- 订单记录数量

如果你已经付款，但软件还显示待支付：

1. 等 10 到 30 秒。
2. 点击账号卡片的 `同步`。
3. 如果还不变，点 `复制订单` 或 `复制诊断` 给维护者。

---

## 12. 隐私模式

默认会隐藏敏感信息。

隐藏内容包括：

- 姓名
- 编号
- UserId
- 登录标识
- 组织和分组细节

什么时候可以临时显示？

- 你要确认账号是不是本人。
- 你要检查账号有没有导错。
- 你自己本机使用，不准备截图发出去。

看完建议立刻切回隐藏。

---

## 13. 账号管理

每个账号都可以：

- 设置备注：例如“本人”“张三”“备用”。
- 设置标签：例如“主号”“备用”“4人间优先”。
- 复制摘要：给自己核对账号状态。
- 复制诊断：给维护者排查。
- 删除账号：不再使用时移除。

删除账号前会二次确认。如果账号正在运行，程序会先停止再删除。

建议：分发给别人使用前，让他们给自己的账号加备注，否则多人账号很容易看混。

---

## 14. 配置备份

进入 `设置` 页面可以备份配置。

建议在这些时候备份：

- 第一次成功添加账号后。
- 批量添加多个账号后。
- 正式使用前。
- 修改偏好或时间策略前。

如果误删账号，可以尝试恢复上一次配置。

---

## 15. 软件更新

进入 `设置` 页面，找到软件更新。

操作步骤：

1. 点击 `检查更新`。
2. 如果显示有新版本，点击 `下载并安装`。
3. 安装完成后点击 `重启生效`。

如果 GitHub 的更新清单入口访问不稳定，程序会尝试 Gitee 备用清单。正式安装包体积较大，目前仍以 GitHub Releases 下载为准。

如果自动更新失败：

1. 优先点击 `GitHub Releases`。
2. 手动下载最新版安装包；`Gitee` 主要用于查看源码、README 和备用更新清单。
3. 关闭 WeUnix。
4. 运行新版安装包覆盖安装。

## 16. 常见问题

### 16.1 一键自动抓取一直没有结果

按顺序检查：

1. 是否点击了 WeUnix 的 `一键自动抓取`。
2. 是否打开了 PC 客户端。
3. 是否真正进入了业务页面，而不是只停留在二维码或授权页。
4. 是否点击了 `停止并恢复` 后重新试过。
5. 是否网络代理异常。

### 16.2 点击启动后看起来没反应

看账号卡片状态：

- 等待开放：说明设置了未来时间。
- 监控中：说明正在轮询资源。
- 已锁定资源：说明已经创建订单，去看二维码。
- 未抢到：说明尝试结束但没有发现可用资源。

### 16.3 房源数量一直是 0

这不一定是软件错误。

可能原因：

- 系统还没开放。
- 当前账号不在批次内。
- 当前偏好没有匹配资源。
- 服务端接口正常但无可选资源。

### 16.4 已经付款但状态不更新

操作：

1. 等 10 到 30 秒。
2. 点击 `同步`。
3. 点击 `复制订单`。
4. 把订单摘要发给维护者。

### 16.5 网络突然打不开网页

如果你刚用过一键自动抓取：

1. 回到 `导入` 页。
2. 点击 `停止并恢复`。
3. 关闭 WeUnix。
4. 重新打开浏览器测试。

### 16.6 字体看起来糊

检查：

1. Windows 显示缩放是否是 100%、125%、150% 这类常规比例。
2. 显示器是否开了奇怪的锐化或缩放。
3. 是否使用了远程桌面或截图压缩。

---

## 17. 给维护者反馈问题时发什么

不要只说“不行”。

请发：

1. 你点到了哪一步。
2. 页面显示什么状态。
3. 是否已经同步成功。
4. 是否已经体检。
5. 是否已经演练。
6. 账号卡片里的 `复制诊断`。
7. 如果是支付问题，再发 `复制订单`。

不要发：

- 完整登录标识。
- 完整 UserId。
- 支付二维码截图。
- 支付链接。
- 身份证、手机号等个人敏感信息。

## 18. 最短操作口诀

```text
先导入
再同步
再体检
再演练
确认时间和偏好
最后启动
出二维码就微信扫码
出问题就复制诊断
```
