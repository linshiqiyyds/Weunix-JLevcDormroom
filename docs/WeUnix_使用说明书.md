# WeUnix Kismet7 使用说明书

## 1. 软件定位

WeUnix 是一个面向寝室选择流程的本地桌面控制台，用来管理多个账号的登录状态、身份信息同步、资源检查、任务演练、正式启动和诊断日志。

程序不会把配置上传到第三方服务器；账号、偏好、时间和敏感信息显示设置都保存在本机配置文件中。

它的核心优势是通过本地后端直连关键 API，减少小程序 WebView 页面加载、白屏等待和重复跳转。按实测体验，关键流程的体感卡顿可降低约 **82%**；实际效果会受网络、设备性能和目标服务状态影响。

## 2. 启动程序

推荐直接运行 release 版本：

```text
E:\Weunix\desktop\gui\src-tauri\target\release\weunix-desktop.exe
```

启动后请确认：

- 顶部状态为 `Live`
- 顶部端点显示为 `服务端点已配置`
- 左侧导航可以正常切换

如果显示后端离线，请关闭程序后重新打开。

## 3. 推荐使用流程

1. 进入“导入”页面，优先使用“一键自动抓取”添加账号。
2. 添加成功后进入“账号”页面，先点击“同步”，确认当前账号是谁。
3. 同步后执行“体检”，检查登录态、身份信息、批次、资源和偏好匹配。
4. 正式使用前先执行“演练”，确认流程可以走通但不会提交正式动作。
5. 确认无误后，再按需要点击单账号“启动”或总览页“启动全部”。

这个顺序可以降低误操作概率，也能让日志和诊断更清楚。

## 4. 导入账号

### 4.1 小白推荐方式

1. 进入左侧“导入”页面。
2. 点击“一键自动抓取”。
3. 程序会临时接管系统代理，用来捕获 PC 客户端里的登录请求。
4. 在 PC 客户端进入目标功能页面。
5. 捕获成功后，WeUnix 会自动添加账号并恢复代理。

### 4.2 高级手动方式

如果你会使用抓包工具，可以复制包含 `wxopid=o...` 的请求 URL、cURL 或文本，粘贴到导入框。

注意：

- `wxcode` 通常是一次性的。
- `wxce...` 这类 appid 不是账号 opid。
- 如果自动添加失败，导入框会保留捕获到的 opid，方便手动重试。

## 5. 同步个人信息

添加账号后，进入“账号”页面或“总览”页的账号卡片，点击“同步”。

同步完成后，“个人信息”区域会展示：

- 姓名
- 组织
- 分组
- 编号
- 角色
- 批次时间

如果显示“待同步”，说明还没有成功执行同步，或接口没有返回对应字段。

## 6. 体检与演练

### 6.1 体检

体检会检查：

- 自动登录是否通过
- Token 是否获取
- 身份信息是否同步
- 批次是否开放
- 资源接口是否返回
- 当前偏好是否匹配

体检不会提交确认单。

### 6.2 演练

演练会模拟正式流程中的关键步骤，用于判断当前账号、偏好和资源是否可用。

演练不会执行正式提交动作，适合正式启动前确认状态。

## 7. 正式启动任务

在确认同步、体检、演练都正常后，可以点击：

- 单个账号卡片中的“启动”
- 总览页顶部的“启动全部”

如果设置了开放时间，程序会等待到指定时间后开始执行。

现在不需要手写时间格式。进入“总览”或“设置”的“执行策略”区域后，可以选择：

- 立即：保存为空，启动后马上执行。
- 今天：选择今天的小时、分钟、秒。
- 明天：选择明天的小时、分钟、秒。
- 自定义：先选日期，再选小时、分钟、秒。

保存策略后，顶部和策略卡片会显示当前生效时间。

## 8. 敏感信息保护

默认开启敏感信息隐藏：

- 姓名脱敏显示
- 组织、分组、角色、批次显示为“已同步”
- 编号、UserId、openid 只显示前后片段

需要完整核对信息时，可以在账号卡片点击“临时核对”，或在执行策略区域切换“显示完整信息”。查看完成后，建议切回隐藏。

账号卡片中的“复制摘要”会按照当前隐私模式生成文本：隐私模式开启时，摘要也会自动打码。

## 9. 账号管理

每个账号卡片都可以：

- 编辑备注名，例如“本人”“张三”“备用”。
- 编辑标签，例如“主号”“备用”。
- 复制摘要，方便确认当前账号是谁。
- 复制诊断，方便发给维护者排查。
- 删除账号。

删除账号会弹出二次确认。若账号正在运行，删除时会先停止任务，再从本地配置中移除。删除前建议到设置页点一次“备份配置”。

## 10. 诊断与日志

日志页会记录：

- 同步状态
- 体检结果
- 演练结果
- 启动和停止动作
- 接口错误
- 资源返回数量

账号卡片中的“复制诊断”会生成一段干净文本，适合直接发给维护者排查问题。

## 11. 配置备份与恢复

设置页提供：

- 备份配置
- 恢复上一次配置

建议在批量添加账号、修改偏好或正式使用前先备份一次。

恢复配置前，程序会保留当前配置副本，便于回退。

## 12. 软件更新

设置页提供“软件更新”卡片：

- GitHub Releases 作为主源。
- Gitee Releases 镜像作为备用源。
- 检查更新失败时会显示可读原因，并提供手动打开发布页的入口。
- 下载更新包后会进行签名校验，安装完成后按提示重启即可生效。

如果 GitHub 访问不稳定，可以优先查看 Gitee 镜像页。

## 13. 内置文档页

左侧导航新增“文档”页面，里面有快速开始、账号管理、隐私显示、时间策略、异常处理和诊断说明。分发给其他用户时，可以让他们先看这个页面。

## 14. 常见问题

### 一键自动抓取没有捕获

请确认 PC 客户端真的进入了目标功能页面。只打开授权入口或二维码页面，通常无法拿到长期可用的 opid。

### 捕获成功但导入失败

可能原因：

- opid 已失效
- 账号未绑定
- 接口拒绝登录
- 网络或代理异常

可以复制诊断文本给维护者。

### 同步后资源数量为 0

这不一定是程序错误。可能是接口正常，但当前没有可用资源，或偏好条件没有匹配。

### 网络异常

如果使用了一键自动抓取，请点击“停止并恢复”。程序会把系统代理恢复到启动前状态。

### 字体看起来不对

源码仓库不内置商业/系统字体文件。程序会优先使用系统可用的苹方、微软雅黑和系统 UI 字体；如果仍然模糊，检查 Windows 缩放比例和显示器缩放设置。

## 15. 分发给他人前建议

- 确认 release exe 能启动。
- 确认配置中没有你的个人账号。
- 默认开启敏感信息隐藏。
- 附带 `使用说明.txt` 或本说明书。
- 告诉用户遇到问题先复制诊断，不要截图暴露完整敏感信息。
